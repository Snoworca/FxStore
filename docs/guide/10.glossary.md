# 용어집

## A

### AUTO 모드 (AUTO Mode)
커밋 모드의 한 종류. 각 쓰기 연산이 즉시 디스크에 반영됩니다.
기본 설정이며, 데이터 손실 위험이 적지만 대량 쓰기 시 성능이 저하될 수 있습니다.

```java
// AUTO 모드 (기본값)
FxStore store = FxStore.open(path);
map.put(key, value);  // 즉시 디스크에 저장
```

## B

### B-Tree
정렬된 키-값 데이터를 저장하는 균형 트리 자료구조입니다.
FxStore의 NavigableMap과 NavigableSet 내부 구현에 사용됩니다.

### BATCH 모드 (BATCH Mode)
커밋 모드의 한 종류. `commit()` 호출 시에만 변경사항이 디스크에 반영됩니다.
대량 쓰기 시 성능이 좋지만, 크래시 시 마지막 커밋 이후 데이터가 손실될 수 있습니다.

```java
FxOptions options = FxOptions.defaults()
    .withCommitMode(CommitMode.BATCH)
    .build();
FxStore store = FxStore.open(path, options);

map.put(key, value);  // 아직 디스크에 저장되지 않음
store.commit();       // 이제 디스크에 저장
```

## C

### 코덱 (Codec)
객체를 바이트 배열로 직렬화(encode)하고 역직렬화(decode)하는 컴포넌트입니다.
`FxCodec<T>` 인터페이스를 구현합니다.

**내장 코덱**:
- `Long`, `Integer`, `Short`, `Byte`
- `Double`, `Float`
- `String`
- `byte[]`

### 컬렉션 (Collection)
Store 내에 이름으로 식별되는 데이터 구조입니다.
NavigableMap, NavigableSet, List, Deque 중 하나입니다.

### COW (Copy-on-Write)
쓰기 시 복사 전략. 수정 시 기존 데이터를 복사하여 새로운 버전을 생성합니다.
원본은 변경되지 않아 읽기 중인 트랜잭션에 영향을 주지 않습니다.
크래시 일관성 보장의 핵심 메커니즘입니다.

### 커밋 (Commit)
변경사항을 디스크에 영구 저장하는 동작입니다.
AUTO 모드에서는 자동으로, BATCH 모드에서는 `store.commit()` 호출이 필요합니다.

### 크래시 일관성 (Crash Consistency)
시스템 크래시 후에도 데이터가 일관된 상태를 유지하는 속성입니다.
FxStore는 COW를 통해 항상 마지막 성공한 커밋 상태로 복구할 수 있습니다.

## D

### Deque
양방향 큐. 앞/뒤 양쪽에서 추가/제거가 가능합니다.
`java.util.Deque` 인터페이스를 구현합니다.

### 내구성 (Durability)
커밋된 데이터가 시스템 장애에도 손실되지 않는 속성입니다.
- `SYNC`: 즉시 동기화 (안전하지만 느림)
- `ASYNC`: 비동기 동기화 (빠르지만 크래시 시 최근 데이터 손실 가능)

## F

### FxCodec
커스텀 타입의 직렬화/역직렬화를 정의하는 인터페이스입니다.

```java
public interface FxCodec<T> {
    String id();
    int version();
    byte[] encode(T value);
    T decode(byte[] bytes);
    int compareBytes(byte[] a, byte[] b);
    boolean equalsBytes(byte[] a, byte[] b);
    int hashBytes(byte[] bytes);
}
```

### FxOptions
Store 설정을 정의하는 빌더 패턴 클래스입니다.
커밋 모드, 내구성, 페이지 크기 등을 설정합니다.

### FxReadTransaction
읽기 전용 트랜잭션. 시작 시점의 스냅샷을 고정하여 일관된 읽기를 제공합니다.

```java
try (FxReadTransaction tx = store.beginRead()) {
    String value = tx.get(map, key);
}
```

### FxStore
FxStore의 메인 인터페이스입니다.
Store 열기/닫기, 컬렉션 생성/열기, 커밋 제어 등을 제공합니다.

## L

### List
인덱스로 접근하는 순서가 있는 컬렉션입니다.
`java.util.List` 인터페이스를 구현합니다.

## M

### 메모리 모드 (Memory Mode)
디스크 대신 메모리에만 저장하는 모드입니다.
`FxStore.openMemory()`로 열 수 있습니다. 테스트나 임시 저장에 적합합니다.

## N

### NavigableMap
정렬된 키-값 맵. 범위 조회, 탐색 메서드를 지원합니다.
`java.util.NavigableMap` 인터페이스를 구현합니다.

### NavigableSet
정렬된 집합. 범위 조회, 탐색 메서드를 지원합니다.
`java.util.NavigableSet` 인터페이스를 구현합니다.

## P

### 페이지 (Page)
디스크 I/O의 기본 단위입니다. 4K, 8K, 16K, 32K, 64K 중 선택할 수 있습니다.
페이지 크기가 클수록 대용량 데이터에 유리하지만 작은 데이터에는 공간이 낭비됩니다.

## R

### 롤백 (Rollback)
BATCH 모드에서 마지막 커밋 이후 변경을 취소합니다.

```java
store.rollback();
```

## S

### 스냅샷 (Snapshot)
특정 시점의 데이터 상태입니다.
읽기 트랜잭션은 시작 시점의 스냅샷을 참조합니다.

### Store
FxStore의 저장 단위입니다. 하나의 파일 또는 메모리 영역에 해당합니다.
여러 컬렉션을 포함할 수 있습니다.

## T

### 트랜잭션 (Transaction)
일련의 연산을 논리적 단위로 묶은 것입니다.
FxStore는 읽기 트랜잭션(`FxReadTransaction`)을 지원합니다.
쓰기는 AUTO 모드의 자동 커밋 또는 BATCH 모드의 명시적 커밋으로 처리합니다.

## V

### 검증 (Verify)
Store의 무결성을 확인하는 동작입니다.

```java
VerifyResult result = store.verify();
if (result.isOk()) {
    System.out.println("Store is healthy");
}
```

## 약어

| 약어 | 전체 이름 | 설명 |
|-----|----------|------|
| COW | Copy-on-Write | 쓰기 시 복사 전략 |
| CRUD | Create, Read, Update, Delete | 기본 데이터 연산 |
| DDL | Data Definition Language | 컬렉션 생성/삭제 등 구조 변경 |
| FIFO | First-In-First-Out | 선입선출 (Queue) |
| LIFO | Last-In-First-Out | 후입선출 (Stack) |
