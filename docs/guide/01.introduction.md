# FxStore 소개

## FxStore란?

FxStore는 **Java 8 기반 단일 파일 영속 컬렉션 라이브러리**입니다.

Java의 표준 컬렉션 인터페이스(`NavigableMap`, `NavigableSet`, `List`, `Deque`)를 구현하면서, 데이터를 **단일 파일에 영속적으로 저장**합니다.

```java
// 일반 Java 코드처럼 사용
FxStore store = FxStore.open(Paths.get("data.fx"));
NavigableMap<Long, String> users = store.createOrOpenMap("users", Long.class, String.class);

users.put(1L, "Alice");
users.put(2L, "Bob");

store.close();

// 재시작 후에도 데이터 유지
store = FxStore.open(Paths.get("data.fx"));
users = store.openMap("users", Long.class, String.class);
System.out.println(users.get(1L));  // "Alice"
```

## 주요 특징

### Zero-Dependency
외부 의존성이 전혀 없습니다. Java 8 표준 라이브러리만 사용합니다.
JAR 파일 하나만 추가하면 바로 사용할 수 있습니다.

### 단일 파일 저장
모든 데이터가 **하나의 파일**에 저장됩니다.
- 백업: 파일 하나만 복사
- 이동: 파일 하나만 이동
- 삭제: 파일 하나만 삭제

### 크래시 일관성
**COW(Copy-on-Write)** 기반으로 크래시가 발생해도 데이터 일관성이 보장됩니다.
항상 마지막으로 성공한 커밋 상태로 복구됩니다.

### Java Collections API 호환
`NavigableMap`, `NavigableSet`, `List`, `Deque` 인터페이스를 그대로 구현합니다.
기존 Java 코드에 쉽게 통합할 수 있습니다.

## 지원 컬렉션

| 컬렉션 | 인터페이스 | 용도 |
|--------|-----------|------|
| **Map** | `NavigableMap<K, V>` | 키-값 저장, 범위 조회, 정렬 순서 |
| **Set** | `NavigableSet<E>` | 고유 요소, 범위 조회, 정렬 순서 |
| **List** | `List<E>` | 인덱스 기반 접근, 순서 유지 |
| **Deque** | `Deque<E>` | 양방향 큐, Stack/Queue로 활용 |

## 지원 타입 (내장 코덱)

별도 설정 없이 바로 사용할 수 있는 타입:

| 타입 | 설명 |
|------|------|
| `Long` | 64비트 정수 |
| `Integer` | 32비트 정수 |
| `Short` | 16비트 정수 |
| `Byte` | 8비트 정수 |
| `Double` | 64비트 부동소수점 |
| `Float` | 32비트 부동소수점 |
| `String` | UTF-8 문자열 |
| `byte[]` | 바이트 배열 |

커스텀 타입은 `FxCodec` 인터페이스를 구현하여 등록할 수 있습니다.

## 언제 사용하나요?

### 적합한 사용 사례

- **임베디드 데이터베이스**: 별도 서버 없이 데이터 저장
- **로컬 캐시**: 애플리케이션 재시작 후에도 유지되는 캐시
- **설정/상태 저장**: 애플리케이션 설정, 사용자 선호도 저장
- **로그/이벤트 저장**: 시계열 데이터, 이벤트 로그
- **작업 큐**: 영속적인 메시지 큐, 작업 대기열
- **테스트/프로토타입**: 빠른 개발 및 테스트

### 부적합한 사용 사례

- **대규모 분산 시스템**: 단일 파일 기반이므로 분산 환경에 부적합
- **높은 동시성**: 다중 프로세스 동시 쓰기 불가
- **수 TB 이상 대용량**: 단일 파일 크기 제한
- **복잡한 쿼리**: SQL과 같은 복잡한 쿼리 미지원
