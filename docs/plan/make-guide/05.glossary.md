# 용어집 초안

## A

### AUTO 모드 (AUTO Mode)
커밋 모드의 한 종류. 각 쓰기 연산이 즉시 디스크에 반영됨.
기본 설정이며, 데이터 손실 위험이 적지만 대량 쓰기 시 성능 저하.

### Allocator
페이지 할당을 관리하는 내부 컴포넌트.
새 페이지가 필요할 때 다음 사용 가능한 페이지 ID 반환.

## B

### B-Tree
정렬된 키-값 데이터를 저장하는 균형 트리 자료구조.
FxStore의 NavigableMap과 NavigableSet 내부 구현에 사용.

### BATCH 모드 (BATCH Mode)
커밋 모드의 한 종류. `commit()` 호출 시에만 변경사항이 디스크에 반영됨.
대량 쓰기 시 성능이 좋지만, 크래시 시 마지막 커밋 이후 데이터 손실 가능.

## C

### Collection (컬렉션)
Store 내에 이름으로 식별되는 데이터 구조.
NavigableMap, NavigableSet, List, Deque 중 하나.

### Codec (코덱)
객체를 바이트 배열로 직렬화/역직렬화하는 컴포넌트.
`FxCodec<T>` 인터페이스 구현. 내장 코덱 (Long, String 등)과 커스텀 코덱 지원.

### COW (Copy-on-Write)
수정 시 기존 데이터를 복사하여 새로운 버전 생성.
원본은 변경되지 않아 읽기 중인 트랜잭션에 영향 없음.
크래시 일관성 보장의 핵심 메커니즘.

### Commit (커밋)
변경사항을 디스크에 영구 저장하는 동작.
AUTO 모드에서는 자동, BATCH 모드에서는 `store.commit()` 호출 필요.

### Crash Consistency (크래시 일관성)
시스템 크래시 후에도 데이터가 일관된 상태를 유지하는 속성.
FxStore는 COW를 통해 항상 마지막 성공한 커밋 상태로 복구 가능.

## D

### Deque
양방향 큐. 앞/뒤 양쪽에서 추가/제거 가능.
`java.util.Deque` 인터페이스 구현.

### Durability (내구성)
커밋된 데이터가 시스템 장애에도 손실되지 않는 속성.
SYNC (즉시 동기화) 또는 ASYNC (비동기 동기화) 선택 가능.

## F

### FxCodec
커스텀 타입의 직렬화/역직렬화를 정의하는 인터페이스.
`encode()`, `decode()`, `compareBytes()` 등의 메서드 포함.

### FxOptions
Store 설정을 정의하는 빌더 패턴 클래스.
커밋 모드, 내구성, 페이지 크기 등 설정.

### FxReadTransaction
읽기 전용 트랜잭션. 시작 시점의 스냅샷을 고정하여 일관된 읽기 제공.
`store.beginRead()`로 시작, `close()`로 종료.

### FxStore
메인 인터페이스. Store 열기/닫기, 컬렉션 생성/열기, 커밋 제어 등 제공.

## I

### Index (인덱스)
빠른 검색을 위한 자료구조.
FxStore에서는 B-Tree와 OST(Ordered String Table)가 내부적으로 사용됨.

## L

### List (리스트)
인덱스로 접근하는 순서가 있는 컬렉션.
`java.util.List` 인터페이스 구현.

## M

### Memory Mode (메모리 모드)
디스크 대신 메모리에만 저장하는 모드.
`FxStore.openMemory()`로 열기. 테스트나 임시 저장에 적합.

## N

### NavigableMap
정렬된 키-값 맵. 범위 조회, 탐색 메서드 지원.
`java.util.NavigableMap` 인터페이스 구현.

### NavigableSet
정렬된 집합. 범위 조회, 탐색 메서드 지원.
`java.util.NavigableSet` 인터페이스 구현.

## O

### OST (Ordered String Table)
정렬된 키-값 저장 구조. FxStore 내부에서 List와 Deque에 사용.

## P

### Page (페이지)
디스크 I/O의 기본 단위. 4K, 8K, 16K, 32K, 64K 중 선택.
페이지 크기가 클수록 대용량 데이터에 유리하지만 작은 데이터에는 공간 낭비.

### PageCache
자주 사용되는 페이지를 메모리에 유지하는 캐시.
`cacheBytes` 옵션으로 크기 조정.

## R

### Rollback (롤백)
BATCH 모드에서 마지막 커밋 이후 변경을 취소.
`store.rollback()` 호출.

### Root Page
트리 구조의 최상위 페이지.
각 컬렉션은 자체 루트 페이지를 가짐.

## S

### Snapshot (스냅샷)
특정 시점의 데이터 상태.
읽기 트랜잭션은 시작 시점의 스냅샷을 참조.

### Store
FxStore의 단위. 하나의 파일 또는 메모리 영역에 해당.
여러 컬렉션을 포함할 수 있음.

### Superblock
Store 메타데이터를 저장하는 특수 페이지.
버전, 페이지 크기, 루트 페이지 ID 등 포함.

## T

### Transaction (트랜잭션)
일련의 연산을 논리적 단위로 묶은 것.
FxStore는 읽기 트랜잭션(`FxReadTransaction`)만 지원.
쓰기는 AUTO 모드의 자동 커밋 또는 BATCH 모드의 명시적 커밋으로 처리.

## V

### Verify (검증)
Store의 무결성을 확인하는 동작.
`store.verify()` 호출. 손상된 페이지, 불일치 등 감지.

## W

### Write Lock
동시에 하나의 쓰기만 허용하는 잠금.
FxStore 내부에서 자동으로 관리됨.

## 약어

| 약어 | 전체 이름 | 설명 |
|-----|----------|------|
| COW | Copy-on-Write | 쓰기 시 복사 전략 |
| CRUD | Create, Read, Update, Delete | 기본 데이터 연산 |
| DDL | Data Definition Language | 컬렉션 생성/삭제 등 구조 변경 |
| FIFO | First-In-First-Out | 선입선출 (Queue) |
| LIFO | Last-In-First-Out | 후입선출 (Stack) |
| OST | Ordered String Table | 정렬된 문자열 테이블 |
