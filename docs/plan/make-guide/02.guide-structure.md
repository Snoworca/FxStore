## 2. 각 문서 상세 설계

### 2.1 00.index.md
- 전체 가이드 목차
- 난이도별 학습 경로 안내
- 빠른 링크

### 2.2 01.introduction.md
**목적**: FxStore가 무엇인지, 왜 사용하는지 설명

```
1. FxStore란?
   - 단일 파일 영속 컬렉션
   - Java Collections API 호환

2. 주요 특징
   - Zero-dependency
   - COW 기반 크래시 일관성
   - 단일 파일 저장

3. 언제 사용하나?
   - 적합한 사용 사례
   - 부적합한 사용 사례

4. 지원 컬렉션
   - NavigableMap
   - NavigableSet
   - List
   - Deque
```

### 2.3 02.quick-start.md
**목적**: 5분 내 첫 코드 실행

```
1. 의존성 추가 (Gradle/Maven)
2. 첫 Store 열기
3. Map에 데이터 저장
4. 데이터 조회
5. Store 닫기
6. 재시작 후 데이터 확인
```

### 2.4 03.installation.md
**목적**: 상세 설치 가이드

```
1. 요구사항
   - Java 8+
   - Gradle 또는 Maven

2. Gradle 설정
3. Maven 설정
4. IDE 설정 (IntelliJ, Eclipse)
5. 버전 호환성
```

### 2.5 04.core-concepts.md
**목적**: 핵심 개념 이해

```
1. Store와 컬렉션
   - Store = 컬렉션들의 컨테이너
   - 컬렉션 = 이름으로 식별

2. 코덱 (Codec)
   - 직렬화/역직렬화
   - 내장 코덱 목록
   - 커스텀 코덱 개요

3. 커밋 모드
   - AUTO: 즉시 커밋
   - BATCH: 명시적 커밋

4. 스냅샷과 일관성
   - COW 원리
   - 읽기 트랜잭션

5. 파일 구조
   - 단일 파일 저장
   - 페이지 기반 구조
```

### 2.9 08.troubleshooting.md
**목적**: 일반적인 문제 해결

```
1. Store가 열리지 않음
   - 파일 잠금 문제

2. 데이터가 저장되지 않음
   - BATCH 모드에서 commit() 누락

3. 성능 문제
   - 캐시 크기 조정
   - 페이지 크기 조정

4. 메모리 부족
   - 대용량 데이터 처리
```
