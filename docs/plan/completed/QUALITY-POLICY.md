## 품질 기준

### 7가지 필수 기준

모든 Phase는 다음 7가지 기준을 **모두 A+** 달성해야 합니다:

1. **Plan-Code 정합성** (15% 가중치)
2. **SOLID 원칙 준수** (20% 가중치)
3. **테스트 커버리지** (20% 가중치)
4. **코드 가독성** (15% 가중치)
5. **예외 처리 및 안정성** (15% 가중치)
6. **성능 효율성** (10% 가중치)
7. **문서화 품질** (5% 가중치)

### A+ 기준

- **각 기준:** 95점 이상
- **총점:** 모든 기준 A+ 필수 (가중 평균은 참고용)

---

## 타협 금지 사항

### ❌ 금지된 완화 조항

다음 행위는 **절대 금지**됩니다:

1. ❌ **"Phase X에서는 Y 기준을 완화한다"** - 모든 Phase 동일 기준
2. ❌ **"미사용 코드는 커버리지에서 제외한다"** - 작성된 모든 코드 테스트 필수
3. ❌ **"시간 부족으로 테스트 생략"** - 시간은 조정 가능, 품질은 불가
4. ❌ **"다음 Phase에서 보완한다"** - 현재 Phase에서 완료 필수
5. ❌ **"중요하지 않은 기능이라 대충"** - 모든 코드 동일한 엄격성
6. ❌ **"일단 구현하고 나중에 리팩토링"** - 처음부터 완벽하게
7. ❌ **"테스트가 너무 길어진다"** - 품질 > 편의성

### ✅ 허용 사항

다음만 허용됩니다:

1. ✅ **기준 미달 시 개선 후 재평가** - 무한 반복 허용
2. ✅ **일정 지연** - 품질 달성 위해 시간 연장 가능
3. ✅ **코드 재작성** - A+ 달성 위해 전면 재작성 가능
4. ✅ **추가 테스트** - 커버리지 향상 위해 테스트 추가 무제한
5. ✅ **문서 개선** - 명확성 향상 위해 반복 개선

## 회귀 테스트 강제

### 무한 반복 프로세스

```
코드 작성
   ↓
테스트 작성
   ↓
모든 테스트 통과? ──No──→ 코드 개선 ──┐
   ↓ Yes                              │
품질 평가 (7가지 기준)                 │
   ↓                                  │
모든 기준 A+? ──No──→ 개선 후 회귀 테스트 ─┘
   ↓ Yes
다음 Phase 진행
```

**종료 조건:** 7/7 기준 A+ 달성 **만**

## 평가 투명성

### 평가 문서화

모든 평가는 다음을 포함해야 합니다:

1. **정확한 수치** - 추정 금지, 실측만 허용
2. **근거 명시** - 모든 점수에 계산식 첨부
3. **미달 사유** - 무엇이 왜 부족한지 구체적 설명
4. **개선 방안** - 실행 가능한 개선 계획 제시
5. **재평가 기록** - 모든 평가 이력 보존

### 자기 평가 금지

- ❌ "대충 90%는 될 것 같다" - 금지
- ✅ "JaCoCo 리포트: 84.5%" - 허용

## 코드 작성 원칙

### 처음부터 완벽하게

**"일단 돌아가게"는 허용되지 않습니다.**

모든 코드는 작성 시점부터:

- ✅ SOLID 원칙 준수
- ✅ 완벽한 예외 처리
- ✅ JavaDoc 완비
- ✅ 테스트 작성 완료
- ✅ 성능 최적화
- ✅ 가독성 확보

### 기술 부채 금지

- ❌ TODO 주석 (개선 계획만 허용)
- ❌ FIXME 주석
- ❌ HACK 주석
- ❌ 임시 코드
- ❌ 주석 처리된 코드

**예외:** 향후 기능 구현 계획을 위한 TODO (현재 구현은 완벽해야 함)

## 테스트 원칙

### 커버리지 목표

- **라인 커버리지:** 90% 이상 (목표 95%)
- **브랜치 커버리지:** 90% 이상 (목표 100%)
- **Mutation 커버리지:** 80% 이상 (Phase 1 이후)

**미달 시:** 무조건 개선

### 테스트 품질

모든 테스트는:

1. **독립성** - 실행 순서 무관
2. **반복성** - 100번 실행 100번 성공
3. **명확성** - 테스트명만 봐도 의도 파악
4. **빠름** - 단위 테스트 < 100ms
5. **Edge Case** - 경계값 모두 테스트
6. **예외 경로** - 모든 예외 검증

## 문서화 원칙

### JavaDoc 100%

- **모든 public 클래스/메서드:** JavaDoc 필수
- **@param, @return:** 모든 매개변수/반환값 설명
- **예제 코드:** 복잡한 API는 사용 예제 포함

### 계획 문서 정합성

- **Plan → Code 추적성:** 모든 요구사항 구현 확인
- **Code → Plan 추적성:** 모든 구현이 계획에 있는지 확인
- **변경 시 문서 동기화:** 코드 변경 시 문서도 즉시 업데이트

## 성능 원칙

### 최적 복잡도

- **시간 복잡도:** 문제의 이론적 하한 달성
- **공간 복잡도:** 불필요한 복사 금지
- **I/O 효율성:** 버퍼링, 배치 처리 활용

**허용 안 함:** "일단 돌아가게, 나중에 최적화"
