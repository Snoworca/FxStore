## 4. 추가 개선 필요사항

### 4.1 목표 달성을 위한 갭 분석

현재: 92% Instruction, 86% Branch
목표: 95% Instruction, 90% Branch
필요: +3% Instruction (~750 inst), +4% Branch (~86 branches)

### 4.2 개선 우선순위

| 순위 | 대상 | 미커버 | 접근법 | 예상 효과 |
|------|------|--------|--------|-----------|
| 1 | core 에러 경로 | 771 inst, 94 branch | verify/compact 에러 테스트 | +1.5% inst |
| 2 | collection 에러 경로 | 544 inst, 152 branch | iterator ConcurrentModification | +1% inst |
| 3 | btree 분할/병합 | 246 inst, 24 branch | 대용량 삽입/삭제 | +0.5% inst |

### 4.3 다음 단계 작업

1. **FxStoreImpl verify 에러 테스트**: 손상된 데이터 시나리오
2. **FxStoreImpl compactTo 에러 테스트**: 파일 시스템 에러 시뮬레이션
3. **Collection iterator 동시성 테스트**: ConcurrentModificationException 경로
4. **BTree 대용량 삽입/삭제 테스트**: 노드 분할/병합 에러 경로
